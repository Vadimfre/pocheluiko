# Руководство по миграции базы данных

## Проблема

Если вы не можете загрузить аватар или изменить email в профиле, это означает, что в вашей базе данных отсутствуют необходимые колонки.

## Решение

### Вариант 1: Автоматическая миграция при запуске сервера

Просто перезапустите backend сервер:

```bash
cd backend
python app.py
```

При запуске сервер автоматически проверит структуру базы данных и добавит недостающие колонки.

### Вариант 2: Ручная миграция

Если вы хотите применить миграции вручную без перезапуска сервера:

```bash
cd backend
python migrate_db.py
```

Скрипт проверит базу данных и добавит следующие колонки в таблицу `users`:
- `avatar` - путь к файлу аватара
- `bio` - информация о пользователе
- `location` - местоположение пользователя

### Вариант 3: Пересоздание базы данных (ВНИМАНИЕ: удалит все данные!)

Если вы хотите начать с чистой базы данных:

```bash
cd backend
rm ecology.db
python app.py
```

**⚠️ ВНИМАНИЕ:** Это удалит всех пользователей, бронирования и другие данные!

## Проверка результата

После миграции вы должны увидеть сообщения:

```
✓ Добавлена колонка 'avatar' в таблицу users
✓ Добавлена колонка 'bio' в таблицу users
✓ Добавлена колонка 'location' в таблицу users
✓ База данных успешно инициализирована
```

## Структура обновленной таблицы users

После миграции таблица `users` будет содержать следующие поля:

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Уникальный идентификатор |
| name | TEXT | Имя пользователя |
| email | TEXT | Email (уникальный) |
| password_salt | TEXT | Соль для хеширования пароля |
| password_hash | TEXT | Хеш пароля |
| favorite_reserve_id | INTEGER | ID любимого заповедника |
| avatar | TEXT | Путь к файлу аватара |
| bio | TEXT | Информация о себе |
| location | TEXT | Местоположение |
| created_at | TEXT | Дата создания аккаунта |

## Функционал после миграции

После успешной миграции вы сможете:

✅ Загружать аватары (JPG, PNG, GIF до 5 МБ)
✅ Редактировать информацию "О себе"
✅ Указывать местоположение
✅ Просматривать аватары других пользователей

## Устранение проблем

### Ошибка: "База данных не найдена"

Запустите backend сервер хотя бы один раз:
```bash
cd backend
python app.py
```

### Ошибка: "Permission denied"

Убедитесь, что у вас есть права на запись в папку `backend/`:
```bash
chmod +w backend/ecology.db
```

### Ошибка при загрузке аватара

Проверьте, что папка `backend/uploads/avatars/` существует и доступна для записи:
```bash
mkdir -p backend/uploads/avatars
chmod 755 backend/uploads/avatars
```

## Дополнительная информация

- Аватары сохраняются в папке `backend/uploads/avatars/`
- Максимальный размер файла: 5 МБ
- Поддерживаемые форматы: JPG, PNG, GIF
- Email нельзя изменить после регистрации (это сделано для безопасности)

## Контакты

Если у вас возникли проблемы с миграцией, проверьте логи сервера или создайте issue в репозитории проекта.
